Why We Don't Teach Testing

http://tinyurl.com/swc-testing-post (inspired by Lorena Barba)

  1. We don't know what tests to write in different domains
  2. We don't know what tolerances to use in those tests

For Example

Steve Eddins: if more than 1% of pixels have changed by more than the low bit in any channel after a refactoring, start looking for the bug.

For Each Problem Do:

  1. Describe the problem
  2. Write a simple solution
  3. Test it (and explain your choices)
  4. Add and test one complexity
  5. Sum up

For Each Problem Don't:

  1. Explain unit testing (we'll do that)
  2. Explain floating point (ditto)
  3. Use math that your colleagues have to look up
